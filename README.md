## 프론트 엔드의 특징

- 효율적인 랜더링을 위해 계층 구조를 가진다.
- 수 많은 컴포넌트가 존재한다.
- 컴포넌트의 수 만큼 처리해야 할 Action이 다양하다.
- 컴포넌트의 수 만큼 모델도 다양하다.

## cqs , cqrs

- 조회랑 명령의 결과는 다르다
- 사이드 이펙트의 유무가 있을것이다.
- model / api, model은 상태, api는 api
